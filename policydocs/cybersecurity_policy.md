# Cybersecurity Policy

## Purpose
Defines controls to protect systems, customer data, and banking operations from cyber threats.

## Security Controls
- Multi-factor authentication for privileged actions
- Network segmentation and endpoint protection
- Secure software development lifecycle practices

## Vulnerability Management
- Continuous scanning for known vulnerabilities
- Risk-based remediation timelines
- Patch management with change control

## Incident Response
- 24x7 monitoring for critical systems
- Defined escalation matrix and communication playbook
- Post-incident review and control improvements

## Access Management
- Joiner/mover/leaver access lifecycle controls
- Periodic access recertification

## Third-Party Risk
- Security due diligence before onboarding
- Contractual security requirements and audits
